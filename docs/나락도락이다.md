
  중요:

  - 아래 코드는 pthread_rwlock_t 의 실제 OS 내부 코드가 아니라
  - 동작 원리를 설명하기 위한 의사코드입니다.

  ———

  # 1. reader lock 의사코드

  pthread_rwlock_rdlock(lock) 을 개념적으로 쓰면:

  function rdlock(lock):
      acquire(lock.internal_mutex)

      while lock.writer_active == true:
          wait(lock.readers_cv, lock.internal_mutex)

      lock.reader_count += 1

      release(lock.internal_mutex)
      return SUCCESS

  의미:

  - writer가 들어와 있으면 reader는 기다림
  - writer가 없으면 reader 수를 1 늘리고 통과

  ———

  # 2. writer lock 의사코드

  pthread_rwlock_wrlock(lock) 을 개념적으로 쓰면:

  function wrlock(lock):
      acquire(lock.internal_mutex)

      lock.waiting_writers += 1

      while lock.writer_active == true or lock.reader_count > 0:
          wait(lock.writers_cv, lock.internal_mutex)

      lock.waiting_writers -= 1
      lock.writer_active = true

      release(lock.internal_mutex)
      return SUCCESS

  의미:

  - reader가 하나라도 있으면 writer는 못 들어감
  - 다른 writer가 있어도 못 들어감
  - 둘 다 없을 때만 writer_active 를 true로 만들고 들어감

  ———

  # 3. unlock 의사코드

  pthread_rwlock_unlock(lock) 은 “지금 unlock 하는 쪽이 reader인지 writer인
  지”에 따라 다르게 동작해야 합니다.

  개념적으로는:

  function unlock(lock, caller_mode):
      acquire(lock.internal_mutex)

      if caller_mode == READER:
          lock.reader_count -= 1

          if lock.reader_count == 0 and lock.waiting_writers > 0:
              signal_one(lock.writers_cv)

      else if caller_mode == WRITER:
          lock.writer_active = false

          if lock.waiting_writers > 0:
              signal_one(lock.writers_cv)
          else:
              broadcast_all(lock.readers_cv)

      release(lock.internal_mutex)
      return SUCCESS

  ———

  # 4. 이 의사코드에서 필요한 내부 상태

  즉 rwlock은 개념적으로 이런 데이터를 갖고 있어야 합니다.

  struct RWLock {
      mutex internal_mutex
      condition_variable readers_cv
      condition_variable writers_cv
      int reader_count
      bool writer_active
      int waiting_writers
  }

  ———

  # 5. 이걸 현재 코드 흐름에 대응시키면

  우리 코드에서:

  pthread_rwlock_rdlock(&entry->lock);

  은 위 의사코드의 rdlock(lock) 에 해당하고,

  pthread_rwlock_wrlock(&entry->lock);

  은 위 의사코드의 wrlock(lock) 에 해당합니다.

  그리고:

  pthread_rwlock_unlock(&entry->lock);

  은 위 의사코드의 unlock(lock, caller_mode) 에 해당합니다.

  ———

  # 6. 우리 프로젝트 함수에 끼워넣은 의사코드

  ## table_runtime_acquire_read()

  function table_runtime_acquire_read(table_name, out_handle):
      entry = registry에서 table_name 찾기
      if entry 없음:
          return FAILURE

      rdlock(entry.lock)
      out_handle.entry = entry
      return SUCCESS

  ## table_runtime_acquire_write()

  function table_runtime_acquire_write(table_name, out_handle):
      entry = registry에서 table_name 찾기
      if entry 없으면:
          entry 생성

      wrlock(entry.lock)
      out_handle.entry = entry
      return SUCCESS

  ## table_runtime_release()

  ## table_runtime_release()

  function table_runtime_release(handle):
      if handle.entry == NULL:
          return

      unlock(handle.entry.lock)
      handle.entry = NULL

  ———

  # 7. 핵심만 다시 요약

  - reader lock:
      - writer 없으면 reader_count += 1
  - writer lock:
      - reader_count == 0 이고 writer_active == false 일 때만 통과
  - unlock:
      - reader면 reader_count -= 1
      - writer면 writer_active = false
